IDE · Agent Mode

Switch to Agent Mode — stop using plain chat

Agent mode doesn't just suggest code. It edits files, runs terminal commands, installs packages, and fixes errors — all autonomously.

  • Open Copilot Chat → switch dropdown to Agent
  • Describe what you want, not how to build it
  • Creates files, runs tests, fixes failures in a loop
// One prompt in Agent mode:
> "Add a /health endpoint with tests"

✓ Created src/routes/health.ts
✓ Created tests/health.test.ts
✓ Ran npm test — 3 passed

Model Selection

Pick the right model for the job

Click the model picker in VS Code or on GitHub. Match the model to the task.

🏗️ Complex Tasks
  • Claude Opus 4.7 — deep refactors
  • GPT-5.4 — large architecture
⚡ Quick Tasks
  • GPT-5.4 mini — fast & cheap
  • Gemini Flash — high volume
  • Switch models mid-conversation anytime
  • Each model has a premium request multiplier
  • CLI supports BYOK & local models (Ollama)
📖 Models & Billing
“The best model is the one that fits the task.”

Prompt Files

Create reusable prompt files

Store task-specific prompts as Markdown files. Invoke them with / slash command. Shareable across your entire team.

.github/prompts/code-review.md
.github/prompts/add-tests.md
.github/prompts/refactor.md
.github/prompts/deploy-checklist.md
  • One file per workflow: review, deploy, migrate, document
  • Include variables, constraints, and output format
  • Team members use the same prompts for consistency

11/18

IDE Shortcuts

Master @workspace, @terminal, and slash commands

Give Copilot full repo context with @workspace. Use @terminal for CLI help. Fire off instant actions.

@workspace "how does auth work?"
@terminal "why did that test fail?"
/fix → auto-fixes selected code
/tests → generates unit tests
/explain → explains code in detail
/doc → adds documentation

Best Practice

Your repo IS Copilot's brain

Copilot's output quality is proportional to your repo quality. Clean structure, good README, and consistent patterns matter.

❌ Messy Repo
  • No README or docs
  • Inconsistent naming
  • Flat file structure
  • → Weak suggestions
✅ Clean Repo
  • Detailed README.md
  • Consistent conventions
  • Logical folder structure
  • → Precise suggestions
  • Write a solid README — it's the first thing Copilot reads
  • Use Copilot Spaces to share curated context
📖 Copilot Best Practices
“Garbage repo in, garbage suggestions out.”
