How to use GitHub Copilot better than 99% of people

Agent Mode, Cloud Agent, MCP Servers, Copilot SDK, and the features most developers don't know exist.

Tip 01
2/14
⚡
IDE · Agent Mode

Switch to Agent Mode — stop using plain chat

Agent mode doesn't just suggest code. It edits files, runs terminal commands, installs packages, and fixes errors — all autonomously in a loop.

  • Open Copilot Chat → switch dropdown to Agent
  • Describe what you want, not how to build it
  • Creates files, runs tests, fixes failures in a loop
// One prompt in Agent mode:
> "Add a /health endpoint with tests
  and update the README"

// Agent executes:
✓ Created src/routes/health.ts
✓ Created tests/health.test.ts
✓ Updated README.md
✓ Ran npm test — 3 passed
“Chat answers questions. Agent ships features.”
Tip 02
3/14
🤖
Cloud Agent

Assign issues directly to Copilot

Copilot Cloud Agent runs in the background on GitHub — no IDE needed. Assign an issue and it researches, plans, codes, and opens a PR.

  • Open any issue → Assignees → select Copilot
  • Also try Claude or Codex agents for comparison
  • Agent signs its commits and runs your CI
  • Works entirely in the cloud — zero IDE time
“Your backlog now has an AI teammate.”

4/14

Config

Add custom instructions — once

Create a single file in your repo. Every Copilot session reads it automatically — your coding standards, stack, and preferences applied every time.

.github/copilot-instructions.md
  • Define language, framework, and testing conventions
  • Set rules like “always use TypeScript strict mode”
  • Org-level instructions now GA (April 2026)
  • Works across IDE, Cloud Agent, and CLI
“Write your rules once. They apply forever.”
Tip 04
5/14
🧠
Model Selection

Pick the right model for the job

Click the model picker in VS Code or on GitHub. Match the model to the task — fast models for quick edits, frontier models for complex architecture.

  • Claude Opus 4.7 — deep multi-file refactors, complex architecture
  • GPT-5.4 mini — fast, cheap, unlimited chats
  • Gemini / Codex — alternatives with different strengths
  • Switch models mid-conversation anytime
“The best model is the one that fits the task.”

6/14
📁
Prompt Files

Create reusable prompt files

Store task-specific prompts as Markdown files. Invoke them with the / slash command in chat. Shareable across your entire team.

.github/prompts/code-review.md
.github/prompts/add-tests.md
.github/prompts/refactor.md
.github/prompts/deploy-checklist.md
  • One file per workflow: review, deploy, migrate, document
  • Include variables, constraints, and output format
  • Team members use the same prompts for consistency
“One great prompt file > 50 random asks.”

MCP Servers

Connect external tools via MCP

Model Context Protocol lets Copilot talk to your databases, APIs, Docker, Azure, and 50+ integrations — mid-conversation.

.vscode/mcp.json
  • Browse the MCP Registry at github.com/mcp
  • Org admins can manage MCP allow-lists for security
  • Copilot reads real data from your systems at runtime
  • No more copy-pasting context into prompts
“Copilot stops being autocomplete. Starts being infrastructure.”
Tip 07
8/14
🖥️
CLI

Copilot CLI — natural language terminal

Run gh copilot from any terminal. Ask in plain English — it generates shell commands, explains them, and executes with your approval.

  • Ask in English, get the exact command
  • Now supports BYOK and local models (April 2026)
  • Activity counts toward your org usage metrics
  • Works on GitHub Mobile too
$ gh copilot suggest "find all files larger than 100MB"

# Suggestion:
find . -type f -size +100M

? Execute command? Yes
“Stop Googling terminal commands.”

9/14
🔍
IDE Shortcuts

Master @workspace, @terminal, and slash commands

Give Copilot full repo context with @workspace. Use @terminal for CLI help. Fire off instant actions without typing paragraphs.

@workspace "how does auth work?"
@terminal "why did that test fail?"
/fix → auto-fixes selected code
/tests → generates unit tests
/explain → explains code in detail
/doc → adds documentation
“Shortcuts beat paragraphs. Every time.”

Code Review

Let Copilot review your pull requests

Copilot Code Review runs on PRs automatically — catches bugs, security issues, and style violations before human reviewers even look.

  • Enable in repo settings → Copilot → Code Review
  • Catches security vulnerabilities automatically
  • PR merge metrics now tracked in usage API
  • Active & passive review user metrics (April 2026)
“AI catches what humans miss. Before merge.”

Best Practice

Your repo IS Copilot's brain

Copilot's output quality is directly proportional to your repo quality. Clean structure, good README, meaningful names, and consistent patterns make every suggestion better.

  • Organize folders logically — Copilot indexes them
  • Write a solid README.md — it's the first thing Copilot reads
  • Use consistent naming conventions throughout
  • Use Copilot Spaces to share curated context with your team
“Garbage repo in, garbage suggestions out.”
