1 Getting Started

Install GitHub Copilot CLI (requires active Copilot subscription)

curl -fsSL https://gh.io/copilot-install | bash

Or via Homebrew / npm:

brew install copilot-cli
npm install -g @github/copilot

Launch & authenticate:

cd your-project
copilot
/login
Scans your codebase and loads custom instructions automatically on start.

4 Copilot CLI Best Practices

  • Run /init first then refine output
  • Be specific in custom instructions
  • Add gotchas Copilot cannot infer
  • Use @filename to include file context
  • Use plan mode for complex features
  • Add path-specific instructions files
  • Keep context clean with /compact
  • Commit to Git for team sharing
  • Use /diff to review before commit
  • Start new session per feature

6 Adding Skills (The Superpower)

Skills = markdown guides Copilot auto-invokes via natural language.

Personal skill ~/.copilot/skills/<name>/

Description field is critical for auto-activation.

8 Setting Up MCP Servers

MCP = Model Context Protocol. GitHub MCP server is built-in.

Built-in Custom Third-party

Add via CLI: /mcp add

{
  "mcpServers": {
    "my-server": {
      "type": "stdio",
      "command": "node",
      "args": ["server.js"],
      "env": {
        "API_KEY": "..."
      }
    }
  }
}

Config: ~/.copilot/mcp-config.json

9 Permissions & Safety

Copilot asks approval before modifying or executing files.

Yes Allow once
Yes + Session Approve for session
No (Esc) Reject & redirect

Trusted Directories

/add-dir /path/to/dir
/list-dirs
/allow-all  # trust everything

Launch Flags

copilot --allow-all
copilot --yolo  # same as above
Inline rejection feedback lets Copilot adapt its approach.
